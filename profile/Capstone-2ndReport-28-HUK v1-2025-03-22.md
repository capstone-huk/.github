# Project-Summary
| 항목 | 내용 |
|:---  |---  |
| (1) 문제 정의 | 현재 전시를 즐긴 관람객 들은 사진과 감상을 함께 저장할 통합적 기록 공간의 분재를 느끼고 있으며 전시를 다녀온 뒤 이를 기념할 무료 기념 요소의 부족을 느끼고 있다. 이에 더해 전국에서 열리는 전시를 원하는 기준에 맞춰 한눈에 파악하기 어려운 상황이며 각자가 즐길 수 있을 전시를 추천 시스템 역시 부재한 상황이다. 이에 서비스의 타켓인 전시를 다녀온 관람객들은 전시 촬영 사진과 감상평을 한번에 관리하며 공유 중심 보다 보관 중심의 전시 기록 플랫폼이 필요하다.  |
| (2) 기존연구와의 비교 | Artsy는 예술 작품을 볼 수 있는 플랫폼으로 작품과 전시를 쉽게 접할 수 있으며 예술 작품 구매 및 경매에 참여가 가능하지만 이는 상업적 측면이 가능하고 예술 작품 구매를 원하는 관람객에만 초점이 맞춰져 있습니다. Google Arts & Culture의 경우 박물관과 갤러리의 전시를 무료로 온라인으로 접근 가능한 플랫폼이지만 한국 지역에 특화된 전시 정보를 제공하고 있지 않을 뿐더라 관람에 대한 기록을 남기는 것 역시 적합하지 않습니다. Instagram과 같은 SNS 채널은 사진과 관람평을 한번에 남길 수 있다는 장점이 있지만 공유 기반의 플랫폼이기에 개인 기록용에 적합하지 않습니다. 이에 'Memoir'는 개인 기록에 중심되어 있으며 전시 탐색부터 사진 및 감상평 업로드에 특화되어 있습니다. |
| (3) 제안 내용 | Memoir은 3가지 기능에 집중하였다. 첫째, 관람객이 다녀온 전시를 기록하는 기능이다. 이는 등록한 전시 리뷰를 조회하고 감상평과 사진을 업로드할 수 있으며 이전 기록들을 한눈에 확인 가능하다. 두번째는 관람객이 전시를 탐색할 수 있는 기능이다. 전시 검색을 통해 주제에 맞는 전시의 상세 정보를 조회할 수 있다. 세번째는 관람객이 관람한 전시를 기념할 수 있는 기능이다. 이를 통해 티켓 굿즈를 생성하고 이전에 생성된 티켓 굿즈들을 조회할 수 있다.<br><br>1. 전시 기록 기능 <br>: 관람객이 다녀온 전시를 기록할 수 있는 기능으로 이전에 다녀왔는 전시의 리뷰를 감상평과 사진과 함께 업로드 할 수 있다. 또한 업로드한 전시들의 리뷰를 상세 조회, 수정, 삭제 가능하며 리뷰를 등록한 전시 리스트들 역시 조회 가능하다.<br><br>2. 전시 탐색 기능 <br>: 관람객이 가고싶은 전시를 탐색할 수 있다. 전시를 검색하고 전시에 대한 상세 정보를 조회하는 것을 통해 전시를 탐색할 수 있다. <br><br>3. 전시 기념 기능<br>: 관람객이 관람했던 전시를 기념할 수 있는 기능으로 다녀왔던 전시 사진을 활용해 티켓 굿즈를 생성하고 이전에 생성했던 티켓 굿즈의 리스트를 조회할 수 있다. |
| (4) 기대효과 및 의의 | Memoir 서비스는 관람객들에게 전시 경험을 기록하고 공유하는 새로운 방법을 제공함으로써 문화 예술에 대한 접근성과 참여도를 높일 것이다. 첫째로, 사용자가 방문한 전시에 대한 리뷰와 감상평을 기록하는 기능은 개인의 문화 활동 이력을 소중하게 보관할 수 있는 디지털 아카이브 역할을 하며, 이는 사용자 각자의 문화적 성장과 기억을 보존하는 데 기여한다. 또한, 이러한 개인적 기록들은 다른 사용자들과의 공유를 통해 커뮤니티 내에서 의미 있는 대화와 인사이트를 촉진시킬 수 있다. <br><br>두번째 기능인 전시 탐색 기능은 사용자가 관심 있는 주제나 장르에 맞는 전시를 쉽게 찾을 수 있도록 도와준다. 이는 문화예술 행사에 대한 정보 접근성을 개선하고, 사용자가 자신의 취향과 관심사에 맞춰 새로운 전시를 발견하고 경험할 수 있는 기회를 넓힌다. 이는 문화예술 분야의 다양성과 포용성을 증진시키는 데에도 기여한다.<br><br>마지막으로, 전시를 기념하는 굿즈 생성 기능은 방문한 전시의 추억을 물리적 형태로 소장할 수 있게 해 준다. 이는 전시 경험을 더욱 특별하게 만들고, 문화 예술 활동을 개인의 일상에 더 깊이 통합시킬 수 있는 방법을 제공한다. 또한, 이 굿즈들은 문화적 대화와 연결의 매개체 역할을 하여 사용자 간, 심지어 세대 간의 문화 교류를 촉진시킬 수 있다.<br><br>전반적으로, Memoir은 문화 예술 활동의 개인적 가치를 높이고, 문화적 경험을 사회적으로 공유할 수 있는 플랫폼을 제공함으로써 문화예술 참여를 활성화시키는 중요한 역할을 할 것이다.|
| (5) 주요 기능 리스트 | <li>카카오 로그인</li><li>전시 리뷰<br>　- 전시 리뷰 등록: 전시 사진 및 관람평 업로드<br>　- 전시 리뷰 상세 조회: 이전에 등록했던 전시 사진과 관람평을 조회<br>　- 전시 리뷰 리스트 조회: 이전에 등록했던 전시 리뷰의 목록을 조회<br>　- 전시 리뷰 수정: 이전 등록했던 전시 리뷰를 수정</li><li>티켓 굿즈<br>　- 티켓 등록: 유저가 리뷰에 등록했던 사진 중 티켓 생성에 사용될 사진을 선택하면 ai를 통해 티켓 굿즈를 생성<br>　- 티켓 굿즈 리스트 조회: 생성한 티켓 리스트를 조회</li><li>전시 검색<br>　- 전시 검색: 키워드를 통한 전시들을 검색 가능<br>　- 전시 상세 조회: 검색된 전시의 위치, 기간, 비용 등 상세 정보를 조회</li>|

StyleShot은 두 개의 이미지를 받아 하나의 생성형 이미지를 출력하는데, style 이미지와 content 이미지를 입력받는다. style 이미지에는 사용자가 선택한 전시작품의 이미지가 들어가고, content 이미지에는 서버에 저장되어있는 추상적인 이미지들이 랜덤하게 들어간다. 이를 조합하여 티켓 굿즈에 사용될 이미지를 생성하게 된다. 이때 Styleshot 모델은 content 이미지의 lineart를 따오고 이를 style 이미지에서 추출한 style로 채워 새로운 이미지를 만든다. 이 모델을 사용하기 위해서 api 배포를 한다. 먼저 해당 모델의 github https://github.com/capstone-huk/AIcode.git 에서 clone을 받고 모델에 필요한 pre-trained weight인 ShtleShot_lineart까지 다운을 받은 뒤, 정리한 파일들을 모아 StyleShot 폴더로 정리한다. 이를 fastapi를 통해 배포하여 우리 프로젝트에서 Styleshot 모델을 사용할 수 있도록 구현하였다._<br><br>~~<br><br>• 역할: 스타일 전이 기반 굿즈 이미지 생성<br>• 기능 설계:<br>　- 사용자 이미지와 전시 스타일 이미지 전달<br>　- 생성된 결과 이미지 수신<br>　- 메타데이터 DB 저장<br>• 방식: 외부 REST API 호출 (JSON + multipart)<br><br>~~**S3 (이미지 스토리지)**<br><br> _사용자가 업로드한 전시 사진을 AWS S3 스토리지에 저장하고, 해당 파일의 URL을 데이터베이스에 저장하여 관리한다. 이 기능의 설계 목적은 대용량 이미지 파일을 서버 외부에 안전하게 저장함으로써 서버의 부담을 줄이고, 추후 이미지 접근을 보다 효율적으로 수행할 수 있도록 하기 위함이다. 기능 구현을 위해 S3Storage 모듈과 URLDatabaseHandler 모듈을 설계하였으며, 이 두 모듈은 이미지의 저장부터 URL 반환, 그리고 URL의 데이터베이스 기록까지 전체적인 흐름을 담당한다. 사용자가 사진을 업로드하면 S3Storage 모듈은 AWS SDK를 이용해 이미지를 S3에 저장하고, 저장된 이미지의 고유 URL을 생성하여 반환한다. 이후 URLDatabaseHandler 모듈이 이 URL을 데이터베이스에 저장하며, 중복 저장을 방지하는 로직과 로그 기록 기능도 함께 포함되어 있다. 이러한 기능의 정상 작동 여부를 검증하기 위해 Postman을 활용하여 실제 전시 이미지를 업로드하고 응답으로 반환된 URL이 S3에 존재하는지 확인하였으며, SQL 쿼리를 통해 해당 URL이 데이터베이스에 정확히 기록되었는지도 점검하였다. 실험 결과, 이미지 파일이 정상적으로 S3에 저장되었고, URL도 DB에 누락 없이 저장되어 본 기능이 의도한 대로 안정적으로 구현되었음을 확인할 수 있었다. 특히 AWS S3를 사용한 이유는 높은 안정성과 확장성, 그리고 정적 리소스에 대한 효율적인 접근을 제공하기 때문이며, 추후 다양한 사용자 요청에도 유연하게 대응할 수 있는 기반이 된다. 이와 같은 설계와 실험 결과는 프로젝트의 핵심 기능 구현이 목적에 부합하게 완료되었음을 보여준다._~~<br><br>• 역할: 사용자 사진 및 생성된 굿즈 이미지 저장소<br>• 기능 설계:<br>　- presigned URL 방식 업로드 처리<br>　- 업로드 후 이미지 URL DB에 저장<br>　- 이미지 접근 경로 사용자에게 제공~~<br><br>**문화포털 전시 조회 API**<br><br>_전시 정보를 외부 공공 데이터 API를 통해 실시간으로 불러와 함께 제공하는 기능을 구현하였다. 이 기능은 사용자에게 보다 풍부하고 정확한 정보를 제공하기 위해 설계되었으며, 국내 공공 전시 데이터 API를 연동하여 현재 진행 중인 전시회의 제목, 장소, 일정, 주최 기관 등의 정보를 가져오는 방식으로 동작한다. API 연동을 위한 테스트는 Postman을 활용하여 수행하였고, API 요청에 대해 JSON 형태로 정확한 응답이 반환되는지 확인하였다. Postman을 통해 실제 데이터를 요청한 결과, 각 전시별 세부 정보가 정상적으로 응답되었으며, 이를 바탕으로 해당 데이터를 가공 및 저장하여 프론트엔드에 노출시킬 수 있도록 구현하였다. 이 과정에서 전시 정보의 누락이나 오류 여부를 반복적으로 검증하였으며, 외부 API 응답 구조를 분석하여 필요한 항목만 선별적으로 추출하고, 사용자 인터페이스에 적합하게 정제하는 로직도 함께 개발되었다. 이러한 설계를 통해 사용자는 전시 사진과 함께 관련된 상세 정보를 직관적으로 확인할 수 있으며, 외부 공공 데이터를 안정적으로 연동하는 기반도 갖추게 되었다._ |
